3.17 Algorithm Efficiency Lesson

Imagine you are doing a research project. You have found a digital library containing every existing published book. You want to find books that contain specific keywords. For example, if you were researching bats, you would probably want to find books containing the keyword "bat". Your computer has a CPU that runs at 3.7 gigahertz.

Assume that the words of every book in that library are sorted alphabetically and stored in a list, and you can use a procedure from an API that takes in an index from that list and returns a book corresponding to the word located at that index. Also assume that the > and < operators work on strings by comparing their alphabetical position. You have two different algorithms that can search that list for a keyword.

Linear Search

PROCEDURE linearSearch(list, value)
{
    index ← 1
    REPEAT UNTIL(index > LENGTH(list))
    {
        IF(list[index] = value)
        {
            RETURN(index)
        }
        index ← index + 1
    }
}

Binary Search

PROCEDURE binarySearch(list, value)
{
    startIndex ← 1
    endIndex ← LENGTH(list)
    REPEAT UNTIL(endIndex < startIndex)
    {
        middleIndex ← startIndex + (endIndex - startIndex) / 2
        middleIndex ← middleIndex - middleIndex % 1
        IF(list[middleIndex] = value)
        {
            RETURN(middleIndex)
        }
        ELSE
        {
            IF(list[middleIndex] > value)
            {
                endIndex ← middleIndex - 1
            }
            ELSE
            {
                startIndex ← middleIndex + 1
            }
        }
    }
    RETURN(-1)
}

Every running of the code segment within the REPEAT UNTIL loops above is considered an "operation".
